### LinkedIn Access & Authentication Research (2026-03-14)

#### LinkedIn's Detection Systems (2026)
- **Session tracking**: Duration, page navigation, scroll behavior, mouse movements
- **IP correlation**: Extensive IP reputation database, datacenter IPs blocked on sight
- **Behavior analysis**: ML models analyze scroll speed, profile reading time, connection patterns
- **Commercial use detection**: High-volume profile views, systematic search crawling, bulk requests
- **Trust Score**: LinkedIn tracks connection acceptance ratio, session patterns

#### LinkedIn Automation Risks
- **30 million accounts restricted in 2025**
- **Permanent bans are almost never overturned** (unlike Instagram)
- **LinkedIn Jail**: 3-14 days for rate limit violations
- **Legal precedent**: hiQ v LinkedIn established public scraping is legal under CFAA, but ToS violations still carry account consequences

#### Best Practices for LinkedIn Automation (2026)

**IP/Proxy Strategy:**
- ✅ **4G/5G mobile proxies**: ~85% account survival rate (best option)
  - Mobile IPs carry legitimate traffic through CGNAT
  - LinkedIn cannot block without disrupting mobile users
  - Natural rotation mimics phone moving between cell towers
- ⚠️ **Residential proxies**: ~50% survival rate (moderate risk)
- ❌ **Datacenter IPs**: Instant ban (avoid completely)

**Rate Limits (2026):**
- Connection requests: 3% of total connections per day
- Messages: Free 50/day, Premium 75/day, Sales Nav 250/day, Recruiter 300/day
- Profile visits: Free 100/day, Premium 250/day, Sales Nav 500/day, Recruiter 600/day
- **20-day withdrawal rule**: Withdraw pending connection requests every 20 days

**Behavioral Guidelines:**
1. **Build foundation first**: 300+ manual connections, authentic engagement baseline
2. **Emulate human behavior**: Random delays, irregular action timing, comfort breaks
3. **Mix activities**: Don't just send requests — likes, profile visits, comments, posts
4. **Avoid mechanical patterns**: Consistency is the biggest red flag
5. **Session fingerprinting**: Vary session duration, don't automate 24/7

**Detection Triggers:**
- Predictable, mechanical session patterns
- Instant action timing (100ms clicks)
- Perfect timing consistency
- High-volume profile views without reading
- Template messages in bulk requests
- API-like access patterns through web interface

#### OpenClaw-Specific Workarounds

**Known Issue: GitHub #12317 (2026.2.6-3)**
- **Problem**: Browser relay detaches on full-page navigation
- **Symptoms**: Tab appears in list, but all CDP commands fail with "tab not found"
- **Cause**: Extension loses CDP target after page reload
- **Current workaround**: Must manually re-click extension icon after navigation
- **Fix status**: No fix yet, requires manual intervention

**Best Approach for LinkedIn:**
1. **Use Chrome Extension Relay** (`profile="chrome"`)
2. **Log in manually first** in your personal Chrome
3. **Attach the tab** with extension icon (badge shows ON)
4. **Stay on same page** — avoid full-page navigations
5. **Add delays**: `browser.act(kind="wait", timeMs=2000)` between actions
6. **If relay disconnects**: Restart browser control server, don't try re-attach

**What NOT to Do:**
- Don't batch 50 actions in 1 second
- Don't check same URL every 5 minutes
- Don't scale to 1000 profile views per hour
- Don't use datacenter IPs or cheap residential proxies
- Don't try to automate 24/7 without breaks
