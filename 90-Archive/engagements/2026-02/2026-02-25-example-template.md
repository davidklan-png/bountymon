# Engagement: Example Structure

This is a template engagement file showing the expected structure.

## Quick Reference

**To create new engagement**:
1. Copy `TEMPLATE.md` to `YYYY-MM-DD-<slug>.md`
2. Fill in sections based on platform and context
3. Generate 3-5 response options with reasoning
4. Present to user in #engagements channel
5. Execute approved response
6. Track metrics and update file

## Channel Commands

In #engagements Discord channel:

```
/engage url:https://...    # Analyze engagement opportunity
/engage history            # Show recent engagements
/engage pending            # List pending proposals
/engage approve <#>        # Approve proposal number
/engage post <#>           # Post approved response
/engage metrics            # Show engagement metrics
```

## Workflow

1. **Discovery**: Identify engagement opportunity (manual or automated)
2. **Analysis**: Fetch context, analyze relevance and risk
3. **Proposal**: Generate 3-5 response options with reasoning
4. **Review**: Present in #engagements for user approval
5. **Execution**: Post approved response
6. **Tracking**: Monitor metrics, schedule follow-ups
7. **Learning**: Update with results and insights

## Status Flow

```
pending → approved → posted → follow-up → completed
```

- **pending**: Awaiting user review
- **approved**: User selected response option
- **posted**: Response published to platform
- **follow-up**: Requires monitoring or additional action
- **completed**: Engagement cycle finished, metrics captured
